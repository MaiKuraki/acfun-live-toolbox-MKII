## 原因
- `showQrLogin()` 在组件初始化阶段被调用时，`generateQrCode` 还是以 `const` 形式声明在后方，处于未初始化状态（TDZ），因此抛出 “Cannot access 'generateQrCode' before initialization”。

## 方案
- 将 `generateQrCode` 改为 `async function generateQrCode()` 的函数声明形式，利用函数声明的提升消除 TDZ 问题，不改动调用位置与逻辑。
- 其余流程保持不变，避免引入新的副作用。

## 修改点
- `packages/renderer/src/components/home/HomeCardAccount.vue`：把 `const generateQrCode = async () => { ... }` 替换为 `async function generateQrCode() { ... }`。

## 验证
- 首页点击“去登录”以及路由 `qrLogin=1` 触发时不再出现初始化错误，弹框正常显示并生成二维码（含自动重试与会话清理）。
