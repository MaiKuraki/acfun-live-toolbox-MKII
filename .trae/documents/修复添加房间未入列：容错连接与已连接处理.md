## 问题
- 添加房间时接口返回“房间已连接”或“获取直播 token 失败”，当前逻辑直接抛错，导致房间未加入列表。
- 预期：即使连接失败或已连接，仍应将房间加入本地列表，并标记为离线或已连接，随后通过 status/details 填充信息。

## 方案
- 修改 `packages/renderer/src/stores/room.ts` 的 `addRoom(roomUrl)`：
  - 调用 `room.connect(roomId)` 后：
    - 若 `success=false && code==='already_connected'` → 视为非致命，继续后续流程。
    - 若其他失败 → 不抛错，记录警告并继续，以离线状态添加房间。
  - 始终构造并添加/更新 `rooms` 项。
  - 始终尝试 `status()` 与 `details()` 以填充数据。
  - 移除最终 `catch` 中的 `throw`，保证流程不会中断。
- 保持现有 “已存在则更新，否则 push” 与 `saveRoomsToStorage()` 行为。

## 验证
- 类型检查通过。
- 行为：
  - 已连接房间再次添加 → 不报错，列表存在该房间。
  - token 获取失败 → 房间加入列表为离线状态，后续刷新可纠正。

## 约束
- 不改动主进程 IPC；仅调整渲染层 store 容错逻辑。
- 不引入新依赖。
