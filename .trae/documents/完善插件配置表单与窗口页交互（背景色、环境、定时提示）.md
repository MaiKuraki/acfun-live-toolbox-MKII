## 目标
- 配置对话框显示可用项：支持“背景色（uiBgColor）”与“定时提示开关（tickerEnabled）”
- 解决 index.js 环境变量获取失效
- 修复定时提示开关点击无响应

## 实施方案
- 配置表单
  - 在示例插件清单中声明表单 schema：
    - `uiBgColor`: type: text, placeholder/default 与示例 UI 一致
    - `tickerEnabled`: type: boolean（用于控制后台定时提示）
  - 位置：
    - `buildResources/plugins/sample-overlay-window/manifest.json` 新增 `config` 字段
    - `buildResources/plugins/sample-overlay-ui/manifest.json` 增加 `tickerEnabled` 项（已有 `uiBgColor`）
  - 说明：`PluginManager` 会合并打包清单默认 config 到用户安装目录（已实现浅合并），配置表单即可显示项

- 窗口页背景色保存无效修复
  - 在 `WindowFramePluginPage.vue` 中已补充 `sendLifecycleEvent('config-updated')`，保存后广播事件，子页收到后 `get-config` 并应用背景色
  - 若仍无效，补充：窗口页保存后主动 `get-config` 一次，确保 UI 立即体现

- index.js 环境变量获取
  - 预加载层已暴露 `plugin.process.execute/message`；完善主进程 IPC 以“按需启动插件进程”：
    - 在 `plugin.process.execute` 处理器中，若进程未运行且清单存在 `main`，自动启动，再执行 `executeInPlugin`
  - 确保窗口页的 `plugin-process` 桥接调用获得成功响应，`getEnvSnapshot()` 返回数据展示在窗口页

- 定时提示开关修复
  - 结合上一步按需启动进程，`getTickerStatus`、`enableTicker/disableTicker` 将返回成功
  - 切换逻辑：点击时先查询状态，再调用对应启停函数；成功后更新按钮文案
  - 后台 3 秒 toast 继续由主进程的 ticker 管理（禁用/卸载插件时清理），窗口关闭不影响

## 验证
- 打开插件管理页面，“配置”对话框显示两项：背景色与定时提示开关；保存后重新打开仍显示已保存值
- 窗口页：保存背景色立即生效；“获取环境变量”按钮显示 Node 版本、内存占用；“定时提示开关”能启停主窗口 toast
- 类型检查：运行 `pnpm typecheck` 确认无误

## 注意
- 不改动 acfunlive-http-api；不使用 mock
- 保持现有桥接与 IPC 通道名一致；只做按需启动进程与配置 schema 声明
