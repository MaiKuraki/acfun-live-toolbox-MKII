## 技术实现
- 渲染层（packages/renderer/src/pages/Settings.vue）：
  - 删除 danmu/plugins/diagnostics 三个 t-tab-panel 与相关状态、方法
  - 通用设置：将 `generalSettings.autoConnect` 改为 `generalSettings.keepLogin`（文案“保持登录”）；移除 `checkUpdates`
  - 网络设置：移除 `networkSettings.useProxy/proxyUrl/timeout/reconnectInterval`，新增 `networkSettings.serverPort`（数值输入）
  - 数据管理：
    - 新增目录选择：使用 `window.electronApi.dialog.showOpenDialog({ properties: ['openDirectory'] })`
    - 打开目录：使用 `window.electronApi.system.showItemInFolder(path)`
    - 导入配置：接受 `.zip`，发送到主进程 IPC 进行解包覆盖
    - 打包导出：发送 IPC 让主进程将当前配置目录打包为 zip 并返回保存路径
  - 保存/加载：通过 `window.electronApi.system.getConfig()/updateConfig()` 读写：
    - `auth.keepLogin: boolean`
    - `server.port: number`
    - `config.dir: string`
- 主进程：
  - ConfigManager（packages/main/src/config/ConfigManager.ts）：
    - 如 electron-store 支持自定义目录，扩展为可传入 `cwd` 或 `name`；新增 `setStorageDir(dir)` 重建 store 到指定目录
    - 若 electron-store 不支持动态切换，采用“配置目录”用于 zip 打包与导入（包含 `store` 文件与 `secrets.json` 等）
  - ipcHandlers（packages/main/src/ipc/ipcHandlers.ts）：
    - 已有 `system.getConfig/system.updateConfig` 用于持久化；新增：
      - `config.setDir(dir)`：设置配置保存目录（调用 ConfigManager 重建或记录）
      - `config.exportZip()`：打包当前配置目录为 zip 并返回路径
      - `config.importZip(filePath)`：解压 zip 覆盖配置目录
    - 复用已存在的 `dialog`/`fs`/`system.showItemInFolder` 能力
  - ApiServer 启动端口（packages/main/src/index.ts）：
    - `const apiPort = configManager.get('server.port', parseInt(process.env.ACFRAME_API_PORT || '18299'))`
    - 监听端口统一为此配置；若更新端口，需触发 `apiServer.stop()` 后以新端口重启（后续可加 IPC `server.restart`）
  - TokenManager（packages/main/src/server/TokenManager.ts）：
    - 读取 `auth.keepLogin`：
      - 为 false 时：`saveTokenInfo()`不写入磁盘；应用退出或登出时清理 `secrets.json`
      - 为 true 时：保持现状（仍持久化）
