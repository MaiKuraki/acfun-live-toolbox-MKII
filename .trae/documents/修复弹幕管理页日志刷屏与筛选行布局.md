## 问题定位
- 日志刷屏来源：渲染进程 `App.vue:29–34` 每 5 秒轮询 `GET /api/acfun/live/stream-status`；同时 `LiveManagePage.vue:402, 540, 560, 899` 内也存在两处 `stream-status` 调用与两个 `setInterval` 定时刷新。
- 服务端打印：`packages/main/src/server/AcfunApiProxy.ts:498–541` 的 `case 'stream-status'` 路由包含多条 `console.log('[AcfunApiProxy][stream-status] ...')`，每次轮询都会打印，导致主控制台出现密集 `[INFO] [main] ...` 与 `[AcfunApiProxy][stream-status] incoming GET`。
- 弹幕筛选行：`LiveDanmuPage.vue` 当前使用 `.filter-row { flex-wrap: wrap; }` 且每个筛选项含 `<label>` 文本 + 控件宽度（180/180/140），在一定窗口宽度下仍可能换行。

## 计划改动
### A. 控制日志与轮询，消除刷屏
1. 渲染层轮询范围收敛：
   - `App.vue:29–34` 改为仅在路由属于 `/live/manage/:id` 或首页需要提示时才轮询；进入其它页面（如弹幕管理 `/live/danmu`）时停止轮询。
   - 将 `pollLiveStatusOnce` 定时器迁移到顶栏或直播管理页专属逻辑，避免全局轮询。
2. 直播管理页去重与节流：
   - 合并 `LiveManagePage.vue` 中两处 `stream-status` 调用，统一到一次初始化 + 一个定时器，确保 `onUnmounted` 清理；减少重复请求。
3. 服务端日志降噪：
   - `AcfunApiProxy.ts:498–541` 将 `stream-status` 相关 `console.log` 降级为 `debug` 或增加条件打印（例如仅在状态变化时打印），避免每次轮询都输出。
4. 保持一致的日志策略：
   - 使用现有 `LogManager` 的级别与最近日志缓存，过滤重复消息（同一内容在 10 秒内仅记录一次）。

### B. 筛选 3 个控件强制同一行
1. UI 布局收敛：
   - 去除每个筛选项的 `<label>`，改用 `placeholder` 展示提示文案，减少水平占用。
   - `.filter-row` 改为 `flex-wrap: nowrap` 且控件设置 `min-width: 0`、`flex: 0 0 auto`，保证在常规宽度下一行展示。
   - 控件宽度保持紧凑（事件类型 180px、关键词 180px、用户过滤 140px），必要时将关键词选择的 tag 折行显示在下一行（现有 `keyword-tags` 保留）。
2. 响应式兜底：
   - 仅在极小屏（<768px）下允许换行，桌面窗口默认不换行。
