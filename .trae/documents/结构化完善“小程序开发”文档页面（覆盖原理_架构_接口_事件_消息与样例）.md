  1) 概览
     - 一图/一卡片：小程序开发整体流程与约束（不使用 mock、真实请求、只读仓库）
  2) 架构（overlay/window/ui/index.js）
     - overlay：单实例与静态托管、SSE 事件与更新
     - window：窗口声明与托管（单实例置顶）
     - ui：只读仓库订阅、总线通信
     - index.js：init/cleanup/handleMessage 约定
  3) 接口（主进程自身 + AcFun HTTP API 代理）
     - 主进程：
       - 事件查询/统计/导出：`GET /api/events`、`GET /api/stats/events`、`GET/POST /api/export`
       - Overlay：`GET /api/overlay/:overlayId`、`POST /api/overlay/:overlayId/action`、`POST /api/overlay/:overlayId/send`、`POST /api/overlay/create`
       - SSE：`/sse/overlay/:overlayId`、`/sse/plugins/:pluginId/overlay`（Last-Event-ID）、`/sse/renderer/readonly-store`、`/sse/system/logs`
       - 包装页：`GET /overlay-wrapper` 与禁用/404 行为
       - 插件静态托管：`/plugins/:id/ui|window|overlay[/*]` 与 `*.html` 直达入口、禁用仅图标例外
     - AcFun 代理（分组列举）：auth/danmu/user/live/gift/manager/badge/replay/preview/image/eventsource（指向外链 apidoc）
  4) 事件
     - Overlay 事件类别与示例：updated/message/closed/action + heartbeat
     - 只读仓库事件：readonly-store-init/update + heartbeat
     - 插件总线：per-plugin overlay 频道消息结构（message/update/closed/action）
  5) 消息传输
     - UI/Window → Overlay（REST 与总线）示例
     - 主进程 → UI/Overlay（SSE 下行）示例
  6) 样例
     - `sample-overlay-ui`：入口 `overlay.html` 与 `index.js` 交互、只读仓库订阅、消息日志
     - `sample-overlay-window`：入口 `window.html` 与 `index.js` 交互、窗口特性
