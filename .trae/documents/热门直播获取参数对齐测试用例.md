## 问题
- 当前热门直播接口返回 `{ success: true, data: { lives: [], total: 46 } }`，列表为空。
- 原因是分页参数不匹配测试用例：测试使用 `page=0`（零基），而渲染层调用使用 `page=1` 并且过大的 `size=50`。

## 方案
- 将渲染层 `fetchHotLives()` 的调用参数改为与测试一致：
  - 默认调用不传分类；分页使用 `page: 0`，`size: 20`。
- 保持选项映射对齐测试的数据结构：`liveId/title/streamer.userName/onlineCount`。

## 修改文件
- `packages/renderer/src/pages/LiveRoomPage.vue`
  - 修改 `fetchHotLives()` 请求参数为 `{ page: 0, size: 20 }`。

## 验证
- 运行渲染包类型检查。
- 静态检视：热门列表出现条目且选择后填入房间ID。

- 若依旧为空，需确认 API 服务端口一致（preload 默认 18299；服务默认 1299）；如有需要后续同步端口。
