**根因定位**
- 主进程使用 electron-store，写入键为点号路径（如 `auth.keepLogin`）。electron-store 会在磁盘上以嵌套对象形式存储（`{ auth: { keepLogin: true } }`）。
- 我们的渲染进程在 `loadSettings()` 里直接通过 `cfg['auth.keepLogin']` 读取，这种方式对嵌套对象无效，导致取到 `undefined`，再经 `!!undefined` 转成 `false`，从而把 UI 恢复成关闭状态。

**解决方案**
1. 在 `Settings.vue` 中实现一个安全的点号路径读取函数：
   - `dotGet<T>(obj, path, defaultValue)`：支持 `a.b.c` 的嵌套读取。
   - `dotBool/ dotNum/ dotStr` 封装不同类型的读取与默认回退。
2. 将 `loadSettings()` 中所有读取改为使用该函数：
   - `keep = dotBool(cfg, 'auth.keepLogin', true)`
   - `minimizeToTray = dotBool(cfg, 'ui.minimizeToTray', false)`
   - `autoStart = dotBool(cfg, 'app.autoStart', false)`
   - `port = dotNum(cfg, 'server.port', 18299)`
   - `dir = dotStr(cfg, 'config.dir', '')`
   - `repo = dotStr(cfg, 'meta.repoUrl', repoUrl.value)`
3. 保持保存逻辑不变（写入点号路径），确保与 electron-store 行为一致。
4. 验证：
   - 打开“保持登录”→显示“设置已自动保存”。
   - 切换到其他页面再回来→仍为开启状态。
   - 其他设置（托盘、自启动、端口、路径、repo）同样保持正确状态。

如确认，我将按上述步骤在 `packages/renderer/src/pages/Settings.vue` 中完成实现与验证。
