## 目标
- 将创建直播页的核心表单字段持久化到本地，并在每次打开页面时自动填充，无需任何手动恢复。

## 持久化范围
- 字段：`basicForm.title`、`basicForm.category`、`basicForm.cover`、`basicForm.mode`。
- 存储介质：`localStorage`（仅采用这一种方式，不保留兼容/回退代码）。
- 存储键：`LIVE_CREATE_BASIC_FORM_V1`。

## 代码位置
- 文件：`packages/renderer/src/pages/LiveCreatePage.vue`。
- 关键引用：
  - 表单数据定义：`basicForm` 在 LiveCreatePage.vue:255-261。
  - 表单使用：标题输入 LiveCreatePage.vue:35，分类级联 LiveCreatePage.vue:49，封面展示 LiveCreatePage.vue:91，模式选择 LiveCreatePage.vue:108。

## 读取与写入时机
- 读取：在 `onMounted` 的最前面调用 `loadDraft()`，先将草稿写入到 `basicForm`，随后执行分类加载与其他初始化；分类加载完成后执行一次 `normalizeCategoryDraft()`，若草稿中的分类不在最新选项中则回退到第一个可用子类。
- 写入：对 `basicForm` 做 `watch`（`deep: true`）监听，发生变动时将当前对象 `JSON.stringify` 后写入 `localStorage`；为避免高频写入，加入轻量节流（如 300ms）。

## 具体实现步骤
1. 在 `<script setup>` 中新增常量与方法：
   - `const DRAFT_KEY = 'LIVE_CREATE_BASIC_FORM_V1'`
   - `function loadDraft()`：读取并解析 JSON，字段类型校验后赋值到 `basicForm`（仅存在的字段覆盖）。
   - `function saveDraft()`：序列化 `basicForm` 并存入 `localStorage`。
   - `function normalizeCategoryDraft()`：根据 `cascaderOptions` 检查 `basicForm.category` 的每级值是否存在；不存在则回退到第一个可用分类/子分类。
2. 在 `onMounted(async () => { ... })` 最前面调用 `loadDraft()`；在 `loadCategories()` 完成后调用 `normalizeCategoryDraft()`。
3. 新增 `watch(basicForm, saveDraft, { deep: true })`，并使用简单节流防抖（例如在 `saveDraft` 内部通过 `setTimeout` 聚合多次变更）。
4. 保持现有封面处理逻辑不变；当封面为 URL 或 base64 时都持久化（封面已有 5MB 限制）。

## 边界与错误处理
- JSON 解析失败时安全忽略并清空草稿。
- 草稿中的分类在最新选项中不存在时做回退，不引入旧版兼容分支。
- 不持久化推流参数、转码状态等运行时信息，仅持久化用户填写项。

## 验证方法
- 打开创建直播页，填写表单但不提交，刷新页面后字段仍自动填充。
- 修改分类选项（服务器返回）后，草稿分类不再存在时回退到第一个可用项。
- 快速频繁输入时不会造成明显卡顿（节流写入）。

## 交付结果
- 修改 `LiveCreatePage.vue` 实现表单草稿持久化与自动填充；不新增文件，不引入回退代码。
