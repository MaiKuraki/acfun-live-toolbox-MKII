## 问题结论
- t-card 的内部 `.t-card__body` 没有固定高度，导致子容器的滚动（`.room-list`）无法出现，从而内容溢出。

## 解决方案
- 在房间列表卡片上为 `.t-card__body` 显式设置高度与弹性：
  - `.room-list-card :deep(.t-card__body) { display:flex; flex-direction:column; min-height:0; height:100%; flex:1; overflow:hidden; }`
- 保持 `.room-list` 为内部滚动：`flex:1; min-height:0; overflow-y:auto;`
- 父级链路已具备高度与伸缩：
  - `.live-room-page { height:100%; }`
  - `.room-list-card { flex:1; min-height:0; }`

## 字体调整
- 将主播名与 UID 的字体调小一级：`.room-streamer { font-size:12px; }`（若当前为 14px）。

## 验证
- 类型检查通过。
- 超过 3 个房间时，卡片内部出现垂直滚动条，内容不再溢出。

## 直播间信息来源（说明）
- 主进程通过 `acfunlive-http-api` 聚合：
  - `live.getUserLiveInfo(userID)` 获得 `liveID/title/onlineCount/liveCover` 等（封面字段来自 `liveCover`）
  - `live.getSummary(liveId)` 获得 `viewerCount/likeCount`
  - `user.getUserInfo(userID)` 获得主播昵称与头像
- 组合后由 `room.details` 返回给渲染层；封面优先用 `liveCover`。
