## 问题判断
- 目前在 `Sidebar.vue` 中使用 `window.electronApi.popup.confirm(...)` 返回值直接赋给变量 `go`，但该桥接返回的是对象 `{ success, result }` 而非布尔值；直接 `if (go)` 并不读取真正的 `result`。
- 使用路由名 `name: 'Home'` 进行跳转可能在某些渲染上下文下解析失败，导致既不跳转也不触发首页卡片的 `qrLogin` 监听。

## 修复方案
1. 规范确认结果的判断：
   - 在 `Sidebar.vue:251-256` 将 `const go = await window.electronApi.popup.confirm(...)` 的返回解析为布尔值：`const resp = await ...; const go = resp?.result === true || resp === true;`。
2. 改用路径跳转并携带查询参数：
   - 将 `router.push({ name: 'Home', query: { qrLogin: '1' } })` 替换为 `router.push({ path: '/home', query: { qrLogin: '1' } })`，避免命名路由解析风险。
3. 保持 QR 弹框联动不变：
   - 首页账号卡片已在 `HomeCardAccount.vue:160-173` 监听 `qrLogin=1`，无需改动。

## 验证
- 未登录点击“创建直播” → 出现双按钮弹窗。
- 点“去登录” → URL 变为 `#/home?qrLogin=1`，首页立即弹出二维码登录框。
- 点“关闭” → 无跳转、无弹框。

## 影响范围
- 仅调整 `Sidebar.vue` 的返回值解析与跳转方式；无需改动弹窗服务、路由结构或首页监听逻辑。
