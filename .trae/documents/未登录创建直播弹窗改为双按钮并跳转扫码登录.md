## 变更目标
- “要直播请先登录”弹窗右下角改为两个按钮：`去登录` 和 `关闭`
- 点击 `去登录`：跳到首页并自动弹出二维码扫码登录框
- 点击 `关闭`：关闭弹窗，无其他动作

## 修改位置
- 替换提示触发点：`packages/renderer/src/components/Sidebar.vue:245-254`
- 支持自定义按钮文案：`packages/renderer/src/services/globalPopup.ts:57-80`

## 交互与路由
- 未登录点击“创建直播”时弹出确认框（双按钮）
- 选择 `去登录`：`router.push({ name: 'Home', query: { qrLogin: '1' } })`，首页监听 `qrLogin=1` 自动调用 `showQrLogin()` 弹出扫码框（已实现于 `packages/renderer/src/components/home/HomeCardAccount.vue:160-173`）

## 实现步骤
1. 在 `GlobalPopup.confirm` 中增加可选参数透传：`confirmBtn`、`cancelBtn`，将其传入 `DialogPlugin.confirm({...})` 用于自定义按钮文本。
2. 在 `Sidebar.vue` 的 `navigateToLiveCreate()` 中：
   - 将 `window.electronApi.popup.alert('提示', '要直播请先登录哦')` 替换为 `window.electronApi.popup.confirm('提示', '要直播请先登录', { confirmBtn: '去登录', cancelBtn: '关闭' })`。
   - 根据返回布尔值：`true` 时执行 `router.push({ name: 'Home', query: { qrLogin: '1' } })`；`false` 时不作为。
3. 删除旧的 `alert` 回退调用，不保留兼容或冗余代码。

## 验证要点
- 未登录点击“创建直播”：弹出双按钮对话框，按钮位于右下角。
- 点击 `去登录`：跳转到首页并立即显示二维码登录对话框。
- 点击 `关闭`：对话框关闭，停留在当前页面。
- 已登录点击“创建直播”：直接进入 `LiveCreate` 页面。

## 影响范围
- 仅影响未登录时的“创建直播”入口弹窗与全局弹窗服务的按钮文案能力；不变更二维码登录流程与路由结构。
