## 目标
- 在主进程控制台明确看到：插件进程使用的入口文件路径，以及工作线程加载插件的方式（ESM/CJS/VM）。
- 不改业务逻辑，只补充日志，快速确认是否仍在用旧版本的 index.js，导致你看不到插件内部日志。

## 修改项（仅日志）
- ProcessManager.startPluginProcess：在实际启动前打印所选 `mainPath` 与 `preferBundled` 标记，格式：`[ProcessManager] starting plugin { pluginId, mainPath, preferBundled }`。
- plugin-worker.js：在 `loadPlugin()` 成功后打印加载方式：`[Worker] plugin loaded { pluginPath, mode: 'esm'|'cjs'|'vm' }`。
- obs-assistant/index.js：模块顶层打印一次：`[obs-assistant] module loaded`（用于确认插件代码为最新且已执行）。

## 验证
- 重启并打开 UI 后，主进程应出现以上三类日志；据此判断是否走了打包版（buildResources）还是用户安装版路径，确认代码是否最新。
- 若路径和加载方式正确但仍无插件内部日志，我将继续沿链路定位（如 getStatus 返回但未输出、循环未启动等）。

## 影响范围
- 仅增加日志输出；不改任何功能或安全策略。
