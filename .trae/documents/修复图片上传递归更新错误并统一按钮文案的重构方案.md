## 问题分析
- 报错：Maximum recursive updates exceeded。根因：当前对 `files` 的 `watch(..., { deep:true })` 中再次赋值 `files.value = [...]`，导致监听源被自身修改→递归触发。
- 目标：消除递归更新；保持按钮文案统一为“重新选择/删除”。

## 重构方案
- 模板
  - 在 `<t-upload>` 增加 `@change="onUploadChange"`，去掉对 `files` 的深度监听。
- 逻辑
  - 删除 `watch(files, ...)` 整段。
  - 新增 `onUploadChange(files)`：
    1) 若存在 `files[0].raw`：读取为 Base64 → 赋值 `basicForm.cover = base64` → 归一化 `files.value = [{ url: base64, name: 原文件名||'cover.jpg', status: 'success' }]`
    2) 若仅有 `files[0].url`：`basicForm.cover = url` → 同样归一化 `files.value` 为 `status:'success'`
    3) 若清空：`basicForm.cover = ''` 且 `files.value = []`
  - 保留 `onMounted` 的草稿回填逻辑（将 `basicForm.cover` 回填为 `files = [{ url, status:'success' }]`）。
- 清理
  - 移除与旧 watch/files 处理相关的冗余代码与注释，遵循“删除回退代码”。

## 验证
- 类型检查：`pnpm -C packages/renderer typecheck`
- 交互验证：
  - 草稿自动带入 → 显示“重新选择/删除”
  - 重新选择图片 → 一次性归一化到 `success`，无递归错误，按钮保持“重新选择/删除”
