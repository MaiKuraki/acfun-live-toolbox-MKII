## 目标
- 无需端口探测，只把插件内部日志（进程检测与连接结果）稳定同步到主进程控制台显示。

## 修改方案
1) 插件日志统一转发到主进程控制台
- 在 `PluginLogger` 的记录方法里（info/warn/error/debug），除了写文件外，追加 `console.<level>` 输出，前缀 `[Plugin <id>]`，确保所有插件日志都会在主进程终端可见。
- 这样即使 worker 的 `plugin_log` 消息未触发，也能通过已有落盘日志进入控制台。

2) 保持现有插件端日志点
- obs-assistant 已在 `isObsRunning`、`connectObs`、轮询循环内打印详细信息（console.info/debug/error），这些会被 logger 捕获并同步到控制台。

## 验证
- 重启后，打开 obs-assistant UI，主进程终端应出现：
  - `[Plugin obs-assistant] [obs-assistant] isObsRunning result { running: ... }`
  - `[Plugin obs-assistant] [obs-assistant] connectObs start ... / success / failed ...`
- 插件管理 → 详情 → 日志与主进程终端显示一致。

## 影响范围
- 仅在日志路径上增加控制台输出，不改业务逻辑与安全设置。
