## 问题现象
- 进入“房间管理”时，已添加房间的按钮与状态出现闪烁（先出现/消失，再恢复）。

## 根因分析
- 页面重复触发加载：`LiveRoomPage.vue` 在 `onMounted` 调用 `roomStore.loadRooms()`，而 `roomStore` 初始化时也会调用 `loadRooms()` 并启动 `startAutoRefresh()`（packages/renderer/src/stores/room.ts:615-620）。→ 进入页面时再次切换 `isLoading`，先渲染 loading，再渲染列表，造成闪动。
- 列表整体替换：`refreshRoomStatus()` 将 `rooms.value` 整体替换为新数组（packages/renderer/src/stores/room.ts:276-280），导致每个房间节点重新挂载，按钮与状态节点重建，出现闪烁。
- 频繁 show/hide：按钮使用 `v-if` 条件与状态快速变化（`status/isLive` 从占位到真实），DOM 销毁/重建引发跳变。
- 全量 CSS 过渡：`.room-item { transition: all 0.2s; }`（LiveRoomPage.vue），所有样式变更都触发动画，加剧视觉闪烁。

## 修复方案
1) 移除页面级重复加载
- 删除 `LiveRoomPage.vue` 的 `onMounted(() => roomStore.loadRooms())`。
- 房间列表依赖全局初始化的 `loadRooms()` 与定时刷新。

2) 列表就地更新，避免整数组替换
- 将 `refreshRoomStatus()` 改为就地更新：按 `room.id` 找到现有项，逐字段赋值（`title/coverUrl/status/isLive/viewerCount/likeCount/streamer` 等），不要重建 `rooms.value` 数组。
- 对于新增或丢失项才执行 push/splice。

3) 降低 DOM 重建频率
- 按钮显示逻辑改为 `v-show` 替代 `v-if`（保持节点存活，仅切换可见性）。
- 采集按钮仅在 `room.isLive` 时 `v-show`，文案依据 `room.status` 三态（connected/connecting/other）。

4) 收敛过渡效果
- 修改 `.room-item` 仅对 `border-color`、`background-color` 做过渡：`transition: border-color .2s, background-color .2s;`，避免布局/尺寸/显示触发全局过渡。

## 验证
- 页面进入时不再出现 loading→列表的闪切；每次刷新只更新文字与样式不重建节点；按钮与状态无闪烁。

## 备注
- 方案仅涉及渲染层（Pinia store + 页面），不改主进程逻辑。完成后仍保持定时刷新与手动刷新按钮可用。
